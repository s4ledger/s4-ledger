SECTION 3: FIELD TYPE REFERENCE
══════════════════════════════════════════════════════════════

status field MUST be one of: "green", "yellow", "red", "pending"
  - green  = on track or completed
  - yellow = late, has open comments/RIDs, or minor issues
  - red    = overdue, blocked, or not submitted past due date
  - pending = under review or coming due (not yet an issue)

responsibleParty MUST be one of: "Government", "Contractor", "Shipbuilder"

received MUST be: "Yes" or "No"

calendarDaysToReview: number (integer) or null (if not yet reviewed)

actualSubmissionDate: "YYYY-MM-DD" string or "" (empty string if not submitted)

dates: All dates use "YYYY-MM-DD" format

contract status MUST be one of: "active", "closeout", "complete"

Title format: "Document Name Rev X (CraftType — Hull N)"
  - The craft type in parentheses MUST match a label in craftRegistry
  - Hull numbers start at 1 and increment per craft type

ids: Programs use "PGM-XXX", Contracts use "CTR-XXX", DRL rows use "DRL-XXX"

contractMapping patterns: lowercase substrings that appear in DRL row titles
  - The app matches row titles to contracts using these patterns
  - Example: if a row title contains "patrol boat", it maps to CTR-001

contracts[].programId: MUST reference a valid program id from programs[]
