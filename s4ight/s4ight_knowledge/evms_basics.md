# Earned Value Management System (EVMS)

## Purpose
EVMS integrates scope, schedule, and cost into a single performance measurement
baseline (PMB). It is mandatory for major DoD acquisition contracts above
DFARS thresholds and is a primary tool for assessing program health.

## Key Metrics
- **PV (Planned Value / BCWS)** — budgeted cost of work scheduled.
- **EV (Earned Value / BCWP)** — budgeted cost of work performed.
- **AC (Actual Cost / ACWP)** — actual cost of work performed.
- **CV** = EV − AC. Negative ⇒ over budget.
- **SV** = EV − PV. Negative ⇒ behind schedule.
- **CPI** = EV / AC. <1.0 ⇒ over budget.
- **SPI** = EV / PV. <1.0 ⇒ behind schedule.
- **EAC** — Estimate at Completion. **VAC** = BAC − EAC.
- **TCPI** — performance required to complete on plan.

## Health Heuristics
- CPI / SPI sustained < 0.95 → likely re-baseline candidate.
- EAC > BAC by >10% with no credible recovery plan → escalate.
- Negative cumulative CV growing in successive months → systemic.

## Integrated Baseline Review (IBR)
- Confirms the PMB is realistic, executable, and integrated with IMS, WBS,
  and risk.
- Conducted post-contract award and after any major re-baseline.

## Common Failure Modes
- IMS not statused, EV measured against a fictional schedule.
- WBS not aligned with contract / SOW; OBS roles unclear.
- Risk reserves missing or unrealistic.
- ILS WBS elements buried in lower levels, invisible at the gate review.

## S4ight Guidance
- For any EVMS question, ask: which contract, which WBS leg (especially
  sustainment / ILS branches), and which reporting month?
- Always recommend tying ILS critical-path activities to specific WBS work
  packages so they appear in CPR/IPMR.
