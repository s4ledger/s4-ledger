# Cybersecurity Sustainment (RMF / ATO / IPS Element 7)

Sustainment-era cybersecurity is owned by Product Support but executed
across PMO engineering, sustaining engineering (ISEA), and the cognizant
authorizing official.

## Frameworks & Authorities

- **DoD RMF** (DoDI 8510.01) — six steps: Categorize → Select → Implement →
  Assess → Authorize → Monitor.
- **ATO / cATO** — Authority to Operate; continuous ATO when monitoring
  meets defined control coverage.
- **eMASS** — authoritative system for RMF artifacts.
- **STIGs** — Security Technical Implementation Guides; applied during
  Implement and re-applied at every baseline update.
- **DoDI 8500.01 / 8140.03** — cybersecurity workforce and roles.

## ILS Hooks (IPS Element 7)

- ISEA owns ongoing patching and STIG re-application schedule.
- TDP must include cyber baseline (ports, protocols, services).
- Training (IPS Element 5) must include role-based cyber training.
- Sustainment funding profile must include cyber maintenance hours.

## Common Failure Modes

- ATO lapses because POA&M closure rate < generation rate.
- STIG re-application is unfunded; configuration drifts after every patch.
- Cybersecurity treated as a one-time event at IOC, not a sustainment
  workstream.

## S4ight Guidance

- For any cyber question, ask: ATO status & expiry, current POA&M count,
  STIG compliance %, last assessment date.
- Tie cyber sustainment to LCSP §7 and the sustaining engineering plan.
- Recommend the PSM include cyber metrics in the monthly PM report.
